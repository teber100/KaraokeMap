# Karaoke Map Wisconsin — Agent Instructions

## Project goal
Build a mobile-first web MVP for finding karaoke events in Wisconsin by date and location.

## Stack
- Next.js
- TypeScript
- Tailwind CSS
- Supabase
- Leaflet
- Vercel

## Product constraints
- This is a tightly scoped MVP
- Public app is read-only
- Only one private admin user for now
- Events are manually curated
- No user accounts beyond admin auth
- No venue or KJ submissions
- No recurring event engine
- No monetization features
- No native mobile app
- No scraping automation

## Coding preferences
- Keep code beginner-friendly and maintainable
- Prefer simple implementations over abstraction
- Use clear file names and folder structure
- Explain where new files should go
- Do not add extra dependencies unless clearly justified
- Do not add features outside the requested scope
- When making schema choices, optimize for clarity and simplicity

## Output expectations
When completing a task:

2. List files created or modified
3. Explain how to run or test the result
4. Call out any follow-up steps
